# ROS2 Research Workspace

A compact ROS 2 workspace containing packages for bringing up and describing
a mobile robot platform. This repository is organized for rapid development
and experimentation with bringup, robot description, and simulation.

## Overview

- **Workspace root:** top-level ROS 2 workspace containing source in `src/`.
- **Primary packages:** [src/my_robot_bringup](src/my_robot_bringup) and
	[src/my_robot_description](src/my_robot_description).

This README documents how to build, run, and contribute to the project.

## Repository layout

- `src/` — package sources (bringup, description, and auxiliary packages)
- `build/`, `install/`, `log/` — workspace build outputs (generated by colcon)

## Prerequisites

- A supported ROS 2 distribution installed (replace `<distro>` below,
	e.g. `humble`, `iron`, or your installed distro).
- `colcon` build tools (install via your distro's instructions or pip).
- Common ROS 2 tooling: `ros2` CLI, `gazebo`/`ignition` if you plan to simulate,
	and standard development tools (cmake, python3).

Example: on Ubuntu/Debian with ROS 2 already installed, you might run

```bash
source /opt/ros/<distro>/setup.bash
python3 -m pip install -U colcon-common-extensions
```

## Quick start — build and source

From the workspace root (this repository):

```bash
# source your ROS 2 distro
source /opt/ros/<distro>/setup.bash

# build the workspace
colcon build --symlink-install

# source the workspace install overlay
source install/setup.bash
```

If you edit Python packages or want faster iteration, keep `--symlink-install`.

## Running the robot (examples)

After sourcing `install/setup.bash`, use `ros2 launch` to start bringup or
visualization. Example commands — adjust launch file names to match the
package's available launch files:

```bash
# Launch a full bringup (bringup launch file may differ)
ros2 launch my_robot_bringup bringup.launch.py

# Launch a visualization or robot_state_publisher using the description
ros2 launch my_robot_description display.launch.py
```

If you use simulation (Gazebo or Ignition), run the appropriate simulator
launch first or use the combined simulator+bringup launch provided by the
bringup package (if present).

## Packages in this workspace

- [src/my_robot_bringup](src/my_robot_bringup): launch files and runtime
	configuration for starting robot nodes, bringing up sensors, and any
	parameter overrides.
- [src/my_robot_description](src/my_robot_description): URDF/Xacro, meshes, and
	related resources for the robot model.

Explore each package for specific launch file names, parameters, and
configuration.

## Development workflow

- Add or change code in `src/`.
- Run `colcon build` from the workspace root.
- Source `install/setup.bash` before running nodes or launch files.

Testing / linting:

```bash
# Run unit tests (if packages include tests)
colcon test --packages-select <package_name>
```

## Contributing

Contributions are welcome. Suggested flow:

1. Create a feature branch from `main` or your preferred branch.
2. Add or update packages under `src/`.
3. Run `colcon build` and any tests locally.
4. Open a pull request with a description of changes and testing steps.

Add a `LICENSE` file to the repository to clarify reuse and distribution.

## Troubleshooting

- If builds fail, remove `build/` and `install/` and rebuild:

```bash
rm -rf build install log
colcon build --symlink-install
```

- Ensure you sourced the correct ROS 2 distro `setup.bash` before building.
- If a launch file isn't found, run `ros2 pkg prefix <package>` to verify
	the package is installed in the overlay.
